# PlaceReviews AI 🗺️

> An intelligent, location-aware place discovery and review platform powered by RAG (Retrieval-Augmented Generation), semantic search, and real-time AI insights.

---

## The Problem

Finding trustworthy, contextually relevant information about places — restaurants, cafes, gyms, shops — is still largely broken. Existing platforms suffer from:

- **Review noise** — thousands of reviews with no intelligent summarization. Users must read through pages of text to form an opinion.
- **No semantic understanding** — search is purely keyword-based. Searching "cozy place for a date" returns nothing useful.
- **Static, non-personalized results** — trending lists are just "most viewed" with no recency bias, activity weighting, or user context.
- **Zero location intelligence** — no distance-aware sorting, no "near me" discovery, no city-aware AI suggestions.
- **Disconnected AI** — most platforms bolt on AI as a gimmick. Here, AI is the core of how you discover, understand, and interact with places.

---

## What I Built

**PlaceReviews AI** solves all of the above. It is a full-stack, production-grade platform where:

- Every place has an **AI-generated summary** synthesized from its real user reviews using RAG
- Users can **ask natural language questions** about any place ("Is it good for families?", "How's the parking?") and get answers grounded in actual reviews
- **Search suggestions** are AI-generated and city-aware — the AI knows you're in Howrah and suggests accordingly
- **Trending places** use a weighted scoring algorithm (views + recent reviews + recent bookmarks + rating) with a 30-day recency window — not just raw view counts
- **Nearby discovery** uses real geolocation, Haversine distance calculation, and bounding-box SQL filtering to find and sort places by actual distance from you
- **Personalized recommendations** are generated by the AI based on your review history and location

## Feature Deep Dive

### RAG-Powered AI (Retrieval-Augmented Generation)
The AI never hallucinates about a place because it only answers based on real, indexed reviews.

**Flow:**
1. When a review is submitted → it is embedded using `all-MiniLM-L6-v2` and stored in ChromaDB
2. When a user requests an AI summary or asks a question → the system retrieves the most semantically similar review chunks from ChromaDB
3. Retrieved chunks are sent as context to the Groq LLM
4. The LLM generates a grounded response — it cannot invent facts not present in reviews

### Trending Algorithm


- Only activity within the last **30 days** counts (configurable)
- Location-filtered: trending near you, not globally
- Falls back to global trending if location returns no results

### Location & Distance
- Browser Geolocation API → coordinates cached for 10 minutes
- Reverse geocoding via OpenStreetMap Nominatim → city name extracted
- Bounding-box SQL pre-filter (cheap) → Haversine post-calculation (accurate)
- Distance badge shown on every place card when location is available
- Dedicated `/api/places/nearby/` endpoint sorted by distance ascending
- All AI features (suggestions, recommendations) are city-aware

## Getting Started

### Prerequisites

- Python 3.11+
- Node.js 18+
- PostgreSQL 14+
- Git

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/place-reviews.git
cd place-reviews
```

### 2. Backend Setup

```bash
cd backend

# Create and activate virtual environment
python -m venv venv

# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### 3. Backend Environment Variables

Create `backend/.env`:

```env
SECRET_KEY=your-django-secret-key-min-32-chars
DEBUG=True

# Database
DATABASE_NAME=place_reviews_db
DATABASE_USER=postgres
DATABASE_PASSWORD=your_password
DATABASE_HOST=localhost
DATABASE_PORT=5432

# Groq AI
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=groq/compound

# RAG / ChromaDB
CHROMA_PERSIST_DIR=./chromadb_data
EMBEDDING_MODEL=all-MiniLM-L6-v2
RAG_DISTANCE_THRESHOLD=0.6
RAG_MIN_RESULTS=1

# Trending algorithm weights (optional — these are defaults)
TRENDING_DAYS_WINDOW=30
TRENDING_VIEW_WEIGHT=1
TRENDING_REVIEW_WEIGHT=3
TRENDING_BOOKMARK_WEIGHT=2
```

> Get a free Groq API key at https://console.groq.com

### 4. Database Setup

```bash
# Create the database in PostgreSQL
psql -U postgres -c "CREATE DATABASE place_reviews_db;"

# Run migrations
python manage.py migrate

# (Optional) Seed with sample data
python manage.py seed_data
```

### 5. Index Reviews into RAG

```bash
# Index all existing reviews into ChromaDB (run once after seeding)
python manage.py index_reviews
```

> ⚠️ Do NOT use the `--analyze` flag — it calls the Groq API for every review and will exhaust your daily token quota.

### 6. Start the Backend

```bash
python manage.py runserver
# API available at http://localhost:8000/api/
```

### 7. Frontend Setup

```bash
cd ../frontend

# Install dependencies
npm install
```

### 8. Frontend Environment Variables

Create `frontend/.env`:

```env
VITE_API_URL=http://localhost:8000/api
```

### 9. Start the Frontend

```bash
npm run dev
# App available at http://localhost:5173
```

## Environment Notes

- The embedding model (`all-MiniLM-L6-v2`) runs **fully locally** — no API calls, no cost
- ChromaDB persists to disk at `CHROMA_PERSIST_DIR` — no external database needed
- Groq free tier gives **500,000 tokens/day** — sufficient for normal app usage
- JWT secret key should be at least 32 characters to avoid security warnings

---

## Known Limitations

- Place data in the demo is seeded with fake addresses (Faker-generated) — real deployment would use a places data provider or user submissions
- Photos are not yet supported in the review submission flow (upload UI coming)
- No email/OTP verification on registration — phone number is stored but not verified
- The `all-MiniLM-L6-v2` model downloads (~90MB) on first run
